Bilateral filter for Smoothing Images

This is my experiment result of smoothing images using bilateral filter.
I copied some code in http://blog.csdn.net/jia20003/article/details/7740683 to implement it. And I modified it to adjust to Android platform. One of the big bug is wrong correspondence which costs me the whole afternoon to figure out. The height of the picture actually refers to the column, not the row. I don’t know why intuitively, but the result tells me that.

A big problem is low efficiency of the program because it handles pixel one by one. For example, if the size of an image is 500500 and the radius is 5(the radius of the program is set to 5 as the image shows). So each pixel will be relative to (52+1)^2 - 1 = 120 nearby pixels. And each step needs k times calculation.So the time will be 500500120*k. It’s hundreds of millions times calculation. No wonder I should wait half a second. I will optimize it.
