matlab gui

Now I have work to do so I don’t have much time to update my blog. I helped to build matlab gui to do modeling automatically and in batch. I have to say that Matlab is easy to use, especially for GUI. It has “guide” and you can create widget by dragging it. All the code work is essentially “set” and “get” attributes of widgets in corresponding callback functions, as you like.

One example is:

To select and read one file:


[filename,filepath]=uigetfile(‘*.xlsx’,’open file’);
filep = strcat(filepath,filename);
[Data,DataText,DataCell]=xlsread(filep);

Sometimes we don’t want to select file one by one manually. We can select one file folder and let Matlab read all files automatically, by using uigetdir.
My code to read all ‘.xlsx’ files in one selected file folder is:


folder_name = uigetdir;
names = dir([folder_name,’*.xlsx’]);
for i = 1:length(names)
[Data,DataText,DataCell]=xlsread([folder_name,’\’,names(i).name]);
end

Varaible “names” is cell array type. Each cell is a struct, so use “names(i).name” to get filename.

In this project, I made some mistakes and found some interesting properties in Matlab, and also there left some problems unsolved.

1.When I create popupmenu in GUIDE, I thought value should correspond to string so I set value to 1 to i. Actually, value will be generated to i automatically, related to ith string. So just set string and leave value default, or popupmenu will not show in figure.

2.Global variables should be declared in all functions/files when you want to use it there. All files in that workspace can visit this variable.

3.


h = gcf;
figure2
close(h);

This code can jump from current figure to figure2. But the location of the figure will change, which appears flash. I do not find better solution for this.

  1. 4.Data is a matrix that contains all numbers, while DataCell is a cell array,whose cell is data in corresponding excel cell. I made a mistake here. I mistook cell array for cell and tried to transform a matrix to cell array using mat2cell, but what I get is actually one cell. Instead, I should use num2cell.
