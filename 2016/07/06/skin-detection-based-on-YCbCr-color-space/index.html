skin detection based on YCbCr color space

Since android face detection api cannot satisfy the requirement of pratical use, I decide to study it by myself. One of direct ways to solve it is from the aspect of skin.

I read some materials and found that YCbCr color space is a good choice. It can be transformed from RGB color space by linear transformation. An easy way is to see whether Cb and Cr is in a certain range. Experiment result shows the range for Cb is 100~120, and that of Cr is 140~170. Since Skin color in Cb and Cr is normally distributed, a more scientific way is to model it by two-dimensional Gaussian Distribution and get probability of each pixel, then set a threshold value to binarize the image. I bypass a step to collect skin color distribution and just use other’s experiment result of mean and covirance matrix S. In most case, it works.

There’s an important problem: illumilation compensation. In my view, it means darken the bright area and brighten the dark area. In YCbCr color space, Y means illumiation, so I calculate the average Y_AVE in the image, for each pixel, get a factor y/Y_AVE and use sigmoid function to get new factor. Use this factor to multiply by r and g and b to darken or brighten the pixel. The reason why I choose sigmoid function is that it is an increasing function and it can let the factor near to 1, because it increases or decreases very slowly. Of course, I do some transformation and adjust constant to make the picture more natural. 

    double y = 0;

for (int row = 0; row < width; ++row){
    for (int col = 0; col < height; ++col){
        pixel = src.getPixel(row, col);    

        y += 0.257*((pixel>>16)&0xff) + 0.564*((pixel>>8)&0xff) + 0.098*(pixel & 0xff) + 16;

    }
}

int wh = width * height;
y /= wh;


//if the whole picture is too dim or bright
f(y < 70){
    y = y/2 + 50;
}else if(y > 130){
    y = y/5 + 100;
}

    for (int row = 0; row < width; ++row){
    for (int col = 0; col < height; ++col){
        pixel = src.getPixel(row, col);    

        int r = (pixel >> 16) & 0xff;  
        int g = (pixel >> 8) & 0xff;  
        int b = pixel & 0xff;

        double y_temp = 0.257*r+0.564*g+0.098*b+16;


        double factor;
        if(y_temp != 0){
            factor = y/y_temp;
        }else{
            factor = 1; 
        }


        //brighten it
        if(factor > 1){ 
            factor = 1 / (1 + Math.pow(Math.E,(-0.7)*(factor - 1))) + 0.5;
        }

        //darken it
        if(factor < 1){
            factor = 1 / (1 + Math.pow(Math.E,(-0.5)*(factor - 1))) + 0.5;
        }


        r = Tools.clamp((int)(r*factor));
        g = Tools.clamp((int)(g*factor));
        b = Tools.clamp((int)(b*factor));

        bmp.setPixel(row,col,Color.rgb(r, g, b));

    }
}

Through this step, the picture will be soft. For example,

the bright area is not that bright now. If the picture is dark, it will be brighter, especially for darker area.

Then I utilize the threshold and probability calculated by two-dimensional Gaussian function to get binary image. In common case, it makes good result.


But the skin sometimes mix up with blond hair, or something similar.

I guess the mean value and covariance matrix of Cb and Cr are not optimal. But I will leave it alone.
